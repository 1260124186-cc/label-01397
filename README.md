# 桂花茶溯源小程序

## How to Run

### 步骤1：安装微信开发者工具
1. 访问微信官方下载页面：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
2. 根据操作系统下载对应版本（Windows 64位/32位 或 macOS）
3. 完成安装后启动微信开发者工具
4. 使用微信扫码登录

### 步骤2：导入小程序项目
1. 打开微信开发者工具
2. 点击左侧「项目」→「导入项目」（或首页「+」号）
3. 在弹出的导入窗口中配置：
   - **目录**：选择本项目文件夹路径
   - **AppID**：填入自己的小程序AppID，或点击「使用测试号」
   - **项目名称**：自动读取为「osmanthus-tea-trace」
4. 点击「导入」按钮

### 步骤3：编译预览
1. 导入成功后，开发者工具会自动编译项目
2. 左侧「模拟器」面板会显示小程序首页
3. 可在模拟器中进行如下测试：
   - 点击「桂花茶扫码溯源」按钮（模拟器中会提示需真机测试）
   - 展开「手动输入溯源ID」，输入 `G001` 或 `G002` 进行查询
   - 查看溯源详情页各模块展示效果

### 步骤4：真机预览/调试
1. 点击工具栏「预览」按钮
2. 用微信扫描生成的二维码
3. 在真机上可完整测试扫码功能

## Services

| 服务模块 | 说明 |
|---------|------|
| 首页 (pages/index) | 品牌展示、扫码溯源、手动输入查询 |
| 详情页 (pages/detail) | 溯源信息完整展示（7大模块） |
| 模拟数据 (utils/mockData) | 本地模拟数据服务，预留后端接口 |
| 扫码服务 | 调用微信原生 wx.scanCode API |
| 区块链存证 | 模拟区块链存证信息展示 |

### 后端接口预留

代码中已预留后端接口调用逻辑，位于 `utils/mockData.js` 和 `pages/detail/detail.js`，可直接替换为真实API调用。

## 题目内容

金色为核心搭配，契合桂花茶调性；顶部展示简约品牌视觉（文字 "一茶一品・桂花茶溯源"+ 桂花图标），整体设计简洁雅致，可点缀小型环保图标（如绿叶）呼应绿色溯源主题。核心交互：突出展示「桂花茶扫码溯源」按钮，点击后调用微信原生扫码API（wx.scanCode），解析二维码中的唯一溯源 ID；增设手动输入溯源ID 功能（支持测试 ID：G001、G002），搭配输入框 + ## 项目介绍

一款聚焦桂花茶全链路溯源场景的微信原生小程序，采用青色+金色为核心色系，实现扫码/手动输入溯源ID查询、溯源详情展示等功能。

### 项目特色

- 🌼 **桂花茶专属**：针对桂花茶特色定制，包含树龄信息、窨制工艺等核心模块
- 🌿 **绿色溯源**：全链路绿色举措展示，涵盖生态种植、环保包装、绿色物流
- 🔗 **区块链存证**：数据真实可查、不可篡改的存证思维
- 🎨 **优雅设计**：青色+金色搭配，契合桂花茶调性

### 功能模块

#### 首页功能
- ✅ 品牌视觉展示（桂花图标 + 品牌名称）
- ✅ 扫码溯源（调用wx.scanCode API）
- ✅ 手动输入溯源ID查询
- ✅ 测试ID快捷填入（G001、G002）
- ✅ 功能说明卡片
- ✅ 区块链存证横幅

#### 溯源详情页模块
1. **基础信息**：溯源ID、批次号、采摘时间、出厂时间
2. **树龄信息**：茶树龄、桂花树龄（核心特色模块）
3. **桂花信息**：品种、产地、采摘时间、香气特点
4. **窨制工艺**：窨制次数/时长/温度、配比、工艺流程图
5. **绿色溯源**：生态种植、环保包装、绿色物流
6. **农残检测**：茶叶+桂花双项检测报告、国标对比
7. **冲泡建议**：水温、时长、续泡次数、小贴士
8. **区块链存证**：链名、区块高度、交易哈希

### 技术规范

- **开发框架**：微信原生小程序
- **基础库版本**：v2.29.0及以上（覆盖99%设备）
- **数据处理**：本地模拟数据，预留后端接口
- **适配要求**：自适应全机型
