## 二、🔴 阻塞级风险（Blocker）

### B-01：URLSearchParams API 兼容性风险

| 项目 | 说明 |
|------|------|
| **风险描述** | 微信小程序环境不完全支持浏览器标准 `URLSearchParams` API，可能导致扫码解析URL格式溯源ID失败 |
| **影响范围** | [index.js](file:///Users/zhangchengcheng/work/ai-project/Trea/solo0605/label-01397/pages/index/index.js#L124-L127) |
| **触发条件** | 用户扫描包含URL参数格式的二维码（如 `https://trace.example.com/query?id=G001`） |
| **严重程度** | 🔴 阻塞级 |
| **修复建议** | 1. 使用兼容方案：手动解析URL参数，避免依赖 `URLSearchParams`；2. 或引入微信小程序兼容的URL解析库；3. 添加 polyfill 兜底处理 |

**问题代码位置**：[parseTraceId](file:///Users/zhangchengcheng/work/ai-project/Trea/solo0605/label-01397/pages/index/index.js#L118-L148)

```javascript
// 问题代码：
if (scanResult.includes('?')) {
  const urlParams = new URLSearchParams(scanResult.split('?')[1]);
  traceId = urlParams.get('id') || urlParams.get('traceId');
}
```

| **修复建议** | 1. 在 `onShareAppMessage` 中添加空值判断；2. 提供默认分享文案 |

**问题代码**：

```javascript
onShareAppMessage: function() {
  const data = this.data.traceData;
  return {
    title: `${data.basicInfo.productName} - 全链路溯源信息`,  // data 可能为 null
```

## 三、🟡 一般级风险（Normal）

| 项目 | 说明 |
|------|------|
| **风险描述** | 多个页面使用 `setTimeout` 模拟延迟，页面卸载后定时器仍可能执行，导致内存泄漏或异常 |
